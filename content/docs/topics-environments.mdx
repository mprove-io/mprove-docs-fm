# Environments

Each project has **prod** environment by default. All project members have access to it.

<Callout type="info" title="Creating additional environments is optional">
  Creating additional environments can be useful for companies that have test
  environments (test databases with the same schema as main database).
</Callout>

A project administrator can create additional environments and give a project member access to that environment.

## Environment Connections

Each connection is tied to one environment.

<Callout type="info">
  Connection names are not unique across environments.
</Callout>

Two or more connections can have the same name but belong to different
environments.

This way, mprove Models can refer to the same connection name in
project files.

Analysts can switch between environments in the UI to test their models.

## Environment Variables

<Callout type="warn">Environment variables are not implemented yet.</Callout>

The basic need for environment variables is solved by using connections that belong to different environments.

If you need environment variables that will change the content of models (for example, table names in models), please let us know (GitHub Issues).
