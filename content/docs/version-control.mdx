# Version Control

## Dev and Production Repositories

Project repositories stores Mprove files:

- [mprove.yml](/docs/ref-project-config) (Project Config)
- [\*.malloy](/docs/ref-model-malloy-source)
- [\*.store](/docs/ref-model-store)
- [\*.chart](/docs/ref-chart)
- [\*.report](/docs/ref-report)
- [\*.dashboard](/docs/ref-dashboard)

When you create a new project you can choose one of two **Remote** options:

- **_Git Clone_** - Mprove connects to your **Remote** repository (Github recommended)
- **_Managed_** - Mprove creates a **Remote** repository locally on the server (not recommended as manual access to such a repository is not supported)

Then Mprove does the following:

- Creates **Production** repository locally on the server that mirrors **Remote** repository content
- Creates Personal **Dev** repository locally on the server for each project member with **FilesEditor** role

Dev repositories allow users to edit files and test changes without affecting **Production** version of files.

Users with **FileEditor** role can manually edit files in their personal Dev repositories.

<Callout type="info">
  To be able to edit files, a user with the **FileEditor** role must select the
  **Dev** repository in the navigation bar.
</Callout>

When user makes a manual change to any file, Mprove does the following:

- Validates all files
- Shows validation errors (if any)
- Updates Models, Charts, Dashboards and Reports that have passed validation

Once the user is satisfied with the changes made, the user can **_commit_** and **_push (deploy)_** changes to **Production** repository to share results with the team.

<Callout type="info">
  When changes are pushed to **Production**, they are actually pushed to
  **Remote** first, and then pulled from **Remote** to **Production**. In other
  words, changes from the **Dev** repository are pushed to the **Production**
  repository via the **Remote** repository in a single user action.
</Callout>

## Git Branches

Small projects may only need to use one branch (usually called master or main).

Advanced git users may work in different branches.

<Callout type="info">
  Users can switch between branches using **repo-branch** selector.
</Callout>

## User Actions

Git related actions are available on the Files page:

- **Save and Validate** - Save the changes. Mprove will automatically check all files in the project repository data folder for errors.

<Callout type="error">
  If errors are found during validation, they need to be fixed so that changes
  can be pushed to production.
</Callout>

- **Commit** - Commit changes in Dev repo.
- **Push** - Deploy committed changes from Dev repo to Production repo (and to Remote repo).
- **Revert to Last Commit** - Discard saved changes in Dev repo that were not committed.
- **Revert to Production** - Revert all changes in the Dev repo to the current Production version of files.
- **Pull from Remote** - Fetch changes from Remote branch and merge into the current branch of Dev repo.
- **Merge branch** - Merge changes from another branch into the current branch.
- **Create branch** - Create a new branch.
- **Delete branch** - Delete current branch.
