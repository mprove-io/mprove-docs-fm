# AI Agent Sessions

## Builder

Builder is a lightweight IDE section of Mprove UI.

Users with File Editor role can browse, edit and validate mprove and malloy files.

There are two types of Builder AI sessions:

- Explorer Session
- Editor Session

## Explorer Session (Under development)

Explorer AI agent can answer user questions by creating draft charts, reports and dashboards based on your mprove models.

## Editor Session

Editor is an AI session that runs in a sandbox (e2b provider).

It takes 10 to 20 seconds for session initial activation time (starting sandbox, pulling user project git repo and starting an opencode server).
It is planned to reduce this time significantly.

After session is activated, there is a safe pause logic in place to save sandbox resources when session is 5 min+ idle.
To resume a paused session (sandbox) you need to send a new message to session agent. Unpause (resume) is fast.
Session state is preserved on pause/resume.

Editor session is powered by open source OpenCode Server with default agents (build/plan + subagents).
You can create and customize project's local opencode config.

To sync uncommitted changes between sandbox repo and mprove server's session repo - AI agent can use "mprove sync" CLI command.

### Editor Session Capabilities

- Setup mprove project
- Add metadata to connection schemas
- Build mprove models
- Build mprove charts, reports and dashboards that are not drafts (using mprove docs)

### Preinstalled in Sandbox template

- node 24.14.0
- brew, curl, git, ripgrep, openssh-client, ca-certificates
- opencode
- [opencode global config](https://github.com/mprove-io/mprove/blob/master/skills/opencode-global-config.json) with mprove-mcp configured
- mprove skills
- mprove cli (only "mprove sync" is needed, other useful commands have corresponding mcp tools)
- mprove docs repository
- malloy docs repository

### Editor Session Credentials

Credentials that are provided to sandbox during session activation: 

- LLM provider api key for selected session Model (see project settings API keys)
- MPROVE_CLI_API_KEY (has access to mprove session repo, branch and env connections through queries)
- MPROVE_CLI_PROJECT_ID
- MPROVE_CLI_HOST
